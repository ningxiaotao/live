痛点与变革

为何不用纯对话框模式？

  • 碎片化交互： 无法一览剧本全局，修改早期剧情需翻找冗长聊天记录。
  • 协同断层： 缺乏多人在线批注和修改痕迹，团队无法高效共创。
  • 上下文易丢失： 难以做到第 100 集与第 1 集人设、设定的精准对照。
03 / 07

1. 灵感生成与扩写

定位：编剧的破冰工具（从 0 到 1）

  • 一键起草： 提供核心想法、人设或大纲，AI 瞬间生成单集/多集剧本草稿。
  • 局部续写： 剧本写到一半卡壳时，基于前文大纲，AI 自动扩写后续剧情。
04 / 07

2. 沉浸式伴写

定位：编剧的隐形副手（智能副编）

  • 光标智能预测： 写作停顿时，AI 自动预测剧情走向，一键采纳补全。
  • 交互式润色： 悬浮菜单提供“更扎心”、“更霸气”等多种重写建议。
  • 选中重构： 选中剧情，直接输入指令（如“加一段嘲讽”），AI 就地改写。
05 / 07

3. 智能质检与协同

定位：剧本的终审判官

  • 实时质检： 一键高亮剧本中的涉黄、血腥、逻辑漏洞及人设偏离。
  • 批注式建议： 审核意见侧边栏展示，一键【接纳】或【忽略】。
  • 团队协同： 支持多人划线批注，AI 根据人类批注提供修改文本。
06 / 07

4. 资产与版本管理

定位：公司的数字资产保险箱

  • 树状目录： 左侧导航栏展示分季、分集，极速点击切换。
  • 无极历史版本： 任何生成与修改均生成快照。
  • 一键回退： 随时回看历史，支持双屏差异比对，确保修改可逆。
07 / 07

开发路径规划

从想法到落地的技术框架

  • 前端： 基于富文本编辑器（如 Quill / Prosemirror）二次开发，加入光标监听。
  • 后端： 封装生成与审计 API，优化长文本 Token 传递策略。
  • 阶段目标： 优先跑通“富文本编辑 + 选词重写 + 侧边栏审计”的 MVP。
